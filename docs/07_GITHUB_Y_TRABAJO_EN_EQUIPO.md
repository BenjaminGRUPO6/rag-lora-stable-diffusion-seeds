# GitHub y trabajo en equipo

## Flujo

1. `main` conserva una versión estable.
2. Cada etapa utiliza una rama.
3. Commits pequeños.
4. Pull Request con revisión de al menos un integrante.
5. No subir datos, modelos ni credenciales.

## Reparto sugerido

- Integrante A: dataset, auditoría y splits.
- Integrante B: clasificador y evaluación.
- Integrante C: LoRA, RAG e integración.
- Todos: revisión, documentación y sustentación.

## Primeras ramas

```text
feature/dataset-acquisition
feature/dataset-audit
feature/vision-baseline
feature/lora-preparation
feature/rag-corpus
```
